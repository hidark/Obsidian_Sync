## 专题 A：打赏 100% vs 订阅分成的 A/B 验证设计

依据 `C1.2` 与 `C3.6`，本项目不预设“订阅一定稀释打赏”或“订阅一定筛选高价值用户”，而是将二者关系定义为运行期可观测结果；其成立前提是 `C1.5` 是否被守住，即订阅是否提供“世界观 + 连载”这类持续内容，而非把原本应打赏获得的即时情绪价值平移进订阅。

### A.1 验证指标定义
PRD 应新增“订阅-打赏关系监控面板”，按创作者、内容期数、用户分层三层查看，核心口径如下：

1. `订阅者打赏 lift`
定义为“订阅者人均打赏金额 / 非订阅者人均打赏金额”。若连续两个观察窗高于 `2.0x`，说明订阅更可能在筛选高意愿用户；若跌破 `1.2x`，说明订阅未显著放大付费深度。

2. `订阅后 7 日打赏率`
定义为用户订阅后 7 天内至少发生一次打赏的占比。该值用于判断订阅是否在创造后续互动场景，而不是一次性替代打赏。

3. `订阅-打赏时间序列协方差`
按周观察订阅收入与打赏收入的创作者级协方差。若大盘协方差持续为正，说明二者偏互补；若多个观察窗为显著负值，需警惕稀释。

4. `订阅内容内打赏渗透率`
定义为发生在“章节页 / 连载页 / 节庆页 / 彩蛋触发页”的打赏笔数，占全部打赏笔数比例。该值直接对应 `C1.5` 是否形成“场景化打赏”。

5. `CTA 暴露占比`
埋点统计创作者端“打赏 CTA 曝光次数 / 订阅 CTA 曝光次数”，以及用户端点击率。若订阅 CTA 明显高曝光且打赏 CTA 在高价值内容节点缺位，则稀释风险上升。

6. `订阅后 ARPPU 增量`
比较同类用户在订阅前后 30 天总付费金额变化。若总付费提升但打赏下降幅度小于订阅新增，说明订阅带来增量；若总付费无提升，仅结构替换，则偏稀释。

7. `高价值用户集中度`
观察创作者收入中前 10% 付费用户贡献占比。若订阅上线后集中度适度提升且打赏 lift 不降，说明在筛选核心用户；若集中度下降且大额打赏减少，则说明高价值行为被弱化。

### A.2 A/B 实验设计
PRD 应规定采用“双层随机 + 分阶段停止”的实验框架。

1. `变量 1：订阅功能 on/off`
以创作者池为随机单位。实验组开放订阅创作者功能，对照组仅保留打赏。该实验回答“订阅是否带来总收入与钱包率增量”，对应 `C1.2`。

2. `变量 2：打赏 CTA 邻近度`
在已开放订阅的创作者内，再随机分为“打赏 CTA 与订阅入口相邻”与“打赏 CTA 远离订阅入口，仅在内容节点出现”。该实验回答“入口设计是否导致机械性替代”。

3. `变量 3：场景化打赏 vs 普通打赏`
在订阅创作者中继续分层：一组提供节日/章节高潮/彩蛋回执等场景化打赏；另一组仅保留普通打赏按钮。该实验直接验证 `C3.6` 所说“筛选成立依赖打赏场景化”。

样本要求：创作者侧至少覆盖 `100-150` 名有效创作者，每组需有足够活跃订阅曝光；用户侧以“每组获得至少 1,000 个首购事件或 300 个订阅事件”为最低读数门槛。实验期建议不少于 `28 天`，覆盖发薪周期与续费周期前半段；若要观察一次续费行为，建议拉到 `42-56 天`。

停止规则应写入 PRD：
- 若实验组总收入提升显著，且打赏收入下降不超过 `15%`，继续放量。
- 若实验组订阅收入增加，但打赏收入下降超过 `25%`，且总收入未显著提升，判为疑似稀释。
- 若“场景化打赏组”显著优于“普通打赏组”，则订阅放量以场景化能力为前置条件，未接入者不得扩量。

### A.3 护栏机制
若运行期监测进入“稀释模式”，应优先动产品机制，而非调整分成，符合 `C1.7`。

警戒阈值建议：
- `订阅者打赏 lift < 1.2x`，连续两周；
- `订阅后 7 日打赏率` 较基线下滑超过 `30%`；
- 订阅收入上涨但创作者总收入未涨，连续两个结算窗；
- 场景内打赏渗透率低于全部打赏的 `25%`。

触发动作：
1. 强制将打赏入口迁回章节高潮、节庆页、完结页，不允许只保留主页订阅入口。
2. 对未配置“世界观/连载”内容结构的创作者暂停订阅流量扶持，落实 `C1.5` 与 `C3.9`。
3. 将订阅购买页文案从“支持创作者”改为“解锁持续内容”，避免侵占打赏心智。
4. 对稀释创作者发出运营整改任务，要求补齐连载节奏与场景化打赏位；未完成前不进入推荐池。

### A.4 打赏场景化产品建议
以下条款可直接入 PRD，作为“筛选模式”的产品前提：

1. `节日/纪念日打赏`
支持创作者配置角色生日、关系纪念日、节日祝福页；打赏后获得专属回执或限定文案。

2. `章节高潮点打赏`
在连载章节的关键节点、完结节点、反转节点展示打赏入口，默认不在普通浏览态高频打断。

3. `订阅者专属礼物`
仅订阅者可购买特定礼物，用于表达更强关系，不替代基础订阅权益。

4. `互动彩蛋回执`
打赏后返回角色专属感谢、隐藏设定、限定图片或下一章预告，强调情绪反馈。

5. `里程碑共创打赏`
当某章节或世界观达到指定进度时开启“解锁下一段设定/番外”的打赏活动，强化参与感而非买断感。

## 专题 B：NSFW 与 SFW 同步上线的支付过渡方案

依据 `C1.6`，NSFW 可售性是发布硬门槛；依据 `C2.3`，iOS 仅允许采用 `方案 A'：完全隐藏 + Web 教育`。因此，同步上线的前提不是“所有端都能买”，而是“Web 可合规成交、iOS 可合规查看已购内容”。

### B.1 发布前依赖清单
PRD 应将以下依赖按“发布硬门槛 / 可并行准备”拆分。

发布硬门槛：
1. 完成 `CCBill/Segpay` 主体申请并获得可交易批复，至少具备一条 NSFW Web 支付链路。
2. 完成 NSFW 创作者 `2257/KYC` 档案归集，未完成者不得开通 NSFW 订阅售卖。
3. 完成高风险商户资料提交与卡组织要求的站点材料准备，包括服务条款、退款政策、年龄声明、内容审核机制。
4. 建立退款、拒付、争议处理 SOP，明确时限、责任人、留证要求。
5. 明确 NSFW/SFW 创作者准入规则与违规下架流程，避免“先卖后审”。

可并行准备但不构成首日阻断：
1. 第二支付通道备用申请，用于主通道风控波动时切换。
2. 争议话术模板、邮件模板、客服 FAQ。
3. 创作者后台的渠道结算说明页与教育文案。

依赖顺序上，应先完成主体申请和站点合规材料，再推进创作者 KYC 归档；因为前者决定能否拿到通道，后者决定哪些供给能上架。

### B.2 iOS 合规方案 A' 的具体落地
PRD 应明确 iOS 内“不可售卖，但允许已购消费”的体验。

用户流程：
1. 用户在 iOS 发现某 NSFW 创作者时，只看到创作者主页、公开简介与可浏览的合规公开内容，不显示“订阅”“开通会员”“价格”按钮。
2. 在主页展示中性教育文案，如“部分内容仅对已开通用户可见”，不得出现站外支付引导词。
3. 用户若尝试查看锁定内容，提示“该内容需在可用渠道完成开通，开通后可在本 App 查看”，不出现外链和购买按钮。
4. 通过站外已触达用户的 Web 页面、邮件、站内信完成教育：订阅需在 Web 完成，完成后返回 iOS 登录同一账号即可查看。
5. 订阅完成后，iOS 内仅负责内容消费、续看提醒与已购身份识别，不负责购买、改价、促销。

这一路径严格对应 `C2.3`，避免在 iOS 内形成支付诱导，同时保留跨端消费闭环。

### B.3 分成按扣渠道费后结算的执行细节
依据 `C1.3`，创作者分成按扣除渠道费后的净收入结算。PRD 应规定后台采用“统一总览 + 渠道拆分明细”的双层展示：

1. 总览层显示“已结算净收入”，避免创作者误以为不同渠道同价必同收。
2. 明细层按 `Web CCBill / Web Segpay / Web Stripe(SFW) / iOS IAP / Android IAP` 分渠道展示订单数、毛收入、渠道费、净收入、创作者分成。
3. 每个渠道旁提供固定说明：“因支付渠道费率不同，同价订阅的可分成金额可能不同”。

是否抹平体验：PRD 应明确“不做全量抹平”。原因是跨渠道长期补贴会把支付成本波动永久留在平台侧，侵蚀 `C1.3` 已确认的毛利逻辑。仅在首发期对重点创作者做限时保底补贴才可行，且须设置期限、名单与预算上限，不应做平台默认规则。

### B.4 发布后 30/60/90 天监测指标
PRD 应要求支付运营设立分阶段健康度看板：

`30 天`：重点看支付可用性
- NSFW Web 支付成功率
- 首购转化率
- 拒付率，目标 `<1%`，预警 `>1.5%`，红线 `>2%`
- 创作者 KYC 完成率与上架通过率

`60 天`：重点看风控稳定性
- 通道账户冻结/限额预警次数
- 争议工单处理时长
- 渠道对账差异率
- 提现成功率与平均到账时长，对应 `C3.5`

`90 天`：重点看经营可持续性
- 渠道结构占比是否过度集中
- NSFW 与 SFW 的退款率差异
- 创作者净收入波动幅度
- 因合规问题导致的下架率、封禁率、申诉率

最终建议写入 Phase 4 结论：NSFW 与 SFW 可同步上线，但前提不是“所有端同时可买”，而是“Web 完成 NSFW 合规成交闭环，iOS 仅承接已购消费”；与此同时，订阅与打赏是否互补，不靠主观判断，而靠 `C1.5 + C3.6` 所定义的运行期指标、实验与护栏共同判定。
