# Phase 4 共识（v2 精简版）：体系博弈 + 支付合规

本 Phase 聚焦两个专题的落地可执行方案，不重复讨论前序已定论议题。

---

## 专题 A：打赏 vs 订阅 A/B 验证设计（C4.A）

### C4.A.1 运行期 7 指标监控面板
PRD 必须内置"订阅-打赏关系监控面板"（按创作者 × 内容期 × 用户分层三层）：

| 指标 | 定义 | 判定线 |
|---|---|---|
| 订阅者打赏 lift | 订阅者人均打赏 / 非订阅者人均打赏 | >2.0x 连续两周 = 筛选；<1.2x = 稀释 |
| 订阅后 7 日打赏率 | 订阅后 7 天内至少一次打赏用户占比 | 基线下滑 >30% = 预警 |
| 订阅-打赏时间序列协方差 | 按周订阅收入与打赏收入协方差 | 持续正 = 互补；多窗口负 = 稀释 |
| 订阅内容内打赏渗透率 | "章节/连载/节庆/彩蛋页"打赏笔数 / 全部打赏 | <25% = 场景化失败 |
| CTA 曝光占比 | 创作者端打赏 CTA vs 订阅 CTA 曝光与点击 | 订阅 CTA 过曝 + 打赏高价值节点缺位 = 稀释风险 |
| 订阅后 ARPPU 增量 | 订阅前后 30 天总付费变化 | 总付费无提升 + 结构仅替换 = 稀释 |
| 高价值用户集中度 | Top 10% 付费用户贡献占比 | 适度提升 + lift 不降 = 筛选；集中度降 + 大额打赏减少 = 稀释 |

### C4.A.2 A/B 实验三变量设计
| 变量 | 实验/对照 | 回答问题 |
|---|---|---|
| 订阅功能 on/off | 创作者池随机分组 | 订阅是否带来总收入与钱包率增量（C1.2） |
| 打赏 CTA 邻近度 | 邻近订阅入口 vs 仅内容节点 | 入口设计是否导致机械替代 |
| 场景化打赏 vs 普通打赏 | 场景化组 vs 按钮组 | 验证 C3.6"筛选成立依赖打赏场景化" |

**样本量**：创作者 100-150 + 每组 1000 首购事件或 300 订阅事件
**实验期**：基础 28 天；观察续费 42-56 天

### C4.A.3 停止规则（写入 PRD）
- 实验组总收入显著提升 + 打赏下降 ≤15% → 继续放量
- 订阅收入涨 + 打赏下降 >25% + 总收入未显著升 → **疑似稀释**
- 场景化打赏组显著优于普通组 → 未接入场景化的创作者不得扩量

### C4.A.4 警戒阈值触发护栏（产品机制而非分成调整）
警戒阈值任一触发：
1. 订阅者打赏 lift <1.2x 连续两周
2. 订阅后 7 日打赏率较基线下滑 >30%
3. 订阅收入涨 + 总收入未涨连续两结算窗
4. 场景内打赏渗透率 <25%

触发动作：
1. 强制打赏入口迁回章节高潮/节庆/完结页
2. 未配置世界观/连载的创作者**暂停订阅流量扶持**
3. 订阅购买页文案从"支持创作者"改为"解锁持续内容"（去除情感替代心智）
4. 稀释创作者发出整改任务，未完成不入推荐池

### C4.A.5 打赏场景化产品 5 条（写入 PRD）
1. **节日/纪念日打赏**：角色生日、关系纪念日、节日祝福页；打赏后专属回执/限定文案
2. **章节高潮点打赏**：连载关键节点/完结/反转点展示入口；不在普通浏览态打断
3. **订阅者专属礼物**：仅订阅者可购，表达更强关系
4. **互动彩蛋回执**：打赏后获角色感谢/隐藏设定/下一章预告/限定图
5. **里程碑共创打赏**：章节/世界观达指定进度时开启"解锁设定/番外"打赏活动

## 专题 B：NSFW 与 SFW 同步上线支付过渡（C4.B）

### C4.B.1 发布硬门槛（C1.6 BLOCKER 补充）
发布前必须完成（任一缺失不能上线）：
1. **CCBill/Segpay 主体批复** + 至少一条 NSFW Web 支付链路
2. **2257/KYC 档案归集**：NSFW 创作者 18+ 身份验证（身份证 + 活体 + 地址）
3. **高风险商户资料提交** + 卡组织要求站点材料（服务条款/退款政策/年龄声明/内容审核）
4. **退款、拒付、争议处理 SOP**：时限/责任人/留证
5. **NSFW/SFW 创作者准入规则 + 违规下架流程**

**可并行准备（非首日阻断）**：
- 第二支付通道备用申请
- 争议话术/邮件/客服 FAQ
- 创作者后台渠道结算说明页

**依赖顺序**：先完成主体申请 + 站点合规，再推进创作者 KYC 归档

### C4.B.2 iOS 方案 A' 的具体 UX 落地（C2.3 执行细则）
用户流程：
1. **iOS 发现 NSFW 创作者** → 仅见创作者主页/公开简介/合规公开内容；**不显示订阅/开通/价格按钮**
2. **主页中性教育文案**："部分内容仅对已开通用户可见"（不得出现站外支付引导词）
3. **尝试查看锁定内容** → 提示"该内容需在可用渠道完成开通，开通后可在本 App 查看"（不出现外链和购买按钮）
4. **通过站外渠道教育**：Web 页面 / Email / 站内信告知"订阅需在 Web 完成，完成后返回 iOS 同账号登录可查看"
5. **订阅完成后**：iOS 内仅内容消费、续看提醒、已购身份识别；不负责购买、改价、促销

### C4.B.3 分成按扣渠道费结算的 UI 展示（C1.3 执行细则）
创作者收入后台采用**"统一总览 + 渠道拆分明细"** 双层展示：
1. **总览层**：显示"已结算净收入"
2. **明细层**：按 `Web CCBill / Web Segpay / Web Stripe(SFW) / iOS IAP / Android IAP` 分渠道展示订单数/毛收入/渠道费/净收入/创作者分成
3. **渠道旁说明**："因支付渠道费率不同，同价订阅的可分成金额可能不同"

**是否抹平 iOS 损耗**：**不做全量抹平**（跨渠道长期补贴会侵蚀 C1.3 确认的毛利逻辑）。仅在**首发期对重点创作者限时保底补贴**（须设期限 + 名单 + 预算上限）。

### C4.B.4 发布后 30/60/90 天监测看板
| 阶段 | 重点 | 关键指标 |
|---|---|---|
| **30 天** | 支付可用性 | NSFW Web 支付成功率；首购转化率；拒付率（目标 <1%，预警 >1.5%，红线 >2%）；KYC 完成率与上架通过率 |
| **60 天** | 风控稳定性 | 通道账户冻结/限额预警次数；争议工单处理时长；渠道对账差异率；提现成功率与平均到账时长 |
| **90 天** | 经营可持续性 | 渠道结构占比是否过度集中；NSFW vs SFW 退款率差异；创作者净收入波动幅度；因合规下架率/封禁率/申诉率 |

### C4.B.5 最终结论
NSFW 与 SFW 可**同步上线**，前提**不是"所有端同时可买"**，而是：
- **Web 完成 NSFW 合规成交闭环**
- **iOS 仅承接已购消费**

订阅-打赏互补/替代判定**不靠主观**，靠 C1.5 + C3.6 + C4.A 定义的**运行期指标/实验/护栏**共同判定。
