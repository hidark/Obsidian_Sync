# Phase 3 共识（v2）：创作者激励与冷启动

## 最终评分（综合 Proposer + Reviewer 对齐）

| 维度 | 评分 | 关键判据 |
|---|---|---|
| **创作者供给质量**（权重 2） | 1 | 头部可启动，腰部窄；承接 C1.5 比例 12-20%（非 Proposer 初估 5-8%），全池 500-1600 人已够用 |
| **与打赏互补**（权重 2） | 1 | 互补性存在但强依赖 C1.5 守住；若订阅退化为图包即转稀释 |
| **生态可持续**（权重 3 生死） | **0.5 + BLOCKER** | 三项同时缺位（无流量扶持 / 无关系可携带 / 无 C1.5 内容形态工具），与 C1.6 并列为 pass/fail 门槛 |
| **支付链路毛利健康**（权重 2） | 1 | 订阅走成人收单+打赏走 IAP 结构整体正向，但 $20 小额首提经济性差 |
| **平台可运营性**（权重 1） | 0.5 | 缺 AI 批量产图审核机制、KYC、诊断看板、订阅者通讯等运营空心 |

**Phase 3 加权总分**：1×2 + 1×2 + 0.5×3 + 1×2 + 0.5×1 = **8 / 10**
**判定**：**部分达成**（生死项未触 0，但 BLOCKER 标记意味着"未补流量扶持 / 关系可携带 / 内容形态工具前不能上线"）

## 达成共识（C3.x 写入 core-commitments）

### C3.1 创作者承接比例重估
- 三件套齐备（世界观+连载+投票）承接率：**5-15%**
- 只要世界观+连载两件套：**15-25%**（C1.5 原意是形态升级，非模块集齐）
- 绝对数量 500-1600 人对 Crushon 已构成有效供给池（类比 OF 早期 100 头部 / FANBOX 早期数千）
- **结论**：不追求广泛点亮，保住 30-80 头部 + 200-500 腰部
- **AI 辅助工具杠杆**（NovelAI Lorebook 类、自动生成设定、剧情分支引擎）可将腰部承接率从 10-20% 提到 20-35%

### C3.2 冷启动 5 大断点（PRD 必补）
1. **首批头部供给错配**：最早开通者里真正会写世界观+规划连载的只占 20-30%
2. **腰部 M2 留存断点**：首月 4-8 订阅 → 次月 2-5 订阅 → 无更新正反馈塌方
3. **AI 批量产图 vs 真实创作难识别**：服务端无法判断世界观"真设定"、连载"真推进"、投票"真参与"
4. **首提中位时长 45-75 天 vs 创作者倦怠窗口 8-12 周重叠**
5. **Crushon 无 Pixiv/DLsite 级分发**：PRD 只给开通/分级/提现，完全无订阅创作者专属曝光位/推荐/榜单/更新 push

### C3.3 升级条件反向激励 + 质量指标必须引入
- 数量型门槛（粉丝数/OC 数/月互动）会成为灰色 ROI 游戏
- **10pp 分成差额 break-even**（含渠道费/iOS 抽成/内部转移）：自购 $100 订阅实回 $75-81，亏 $19-25
- 仅 Web 低价档有明显自购动机
- **必须补充质量指标**：30 日续费率 / 退款率 / 更新达标率 / 订阅后活跃度 / 履约率
- 白名单仅用于冷启动 60-90 天

### C3.4 C1.5 产品形态落地优先级
- **世界观档案**：P0（订阅页新增档案页 + 连载目录 + 时间轴；直接把订阅从"图包"升级到"宇宙入场"）
- **连载承诺**：P0（月更节奏标签 + 最近更新时间 + 下次预计时间；连续两期未达标自动降权取消推荐加权）
- **订阅者投票**：P1（月度选题/剧情分支/关系走向；创造"共同决定"参与感；创作者运营负担较高）

### C3.5 二段式提现规则（修正）
基于 Reviewer 挑战 $20 首提经济性：
| 层级 | 首提门槛 | 常规门槛 | 月次数 | 渠道约束 |
|---|---|---|---|---|
| 新创作者首提 | **≥$30**（仅 1 次，$20 改 $30 因 Wire 成本） | — | — | PayPal / ACH 低费 |
| 长尾（前 60 天） | — | 首提豁免 1 次 | 2 次/月 | — |
| 腰部 | ≥$30 | ≥$50 | 2 次/月 | 标准 |
| 头部/认证 | ≥$30 | ≥$30 | 4 次/月 | 放宽 $2000 月封顶 |

**防刷规则**：首提绑定 KYC + 首提前 30 天订阅者必须非同 IP/同设备/同支付方式的真实用户（退订率 <20%）

### C3.6 打赏 100% 的双重条件判定
| 条件 | Proposer（稀释）成立 | Reviewer（筛选）成立 |
|---|---|---|
| 订阅者打赏 lift | 打赏率 ≈ 非订阅者 | 打赏率 ≥ 2x 非订阅者 |
| 订阅内容稀缺性 | C1.5 失守（图包） | C1.5 守住（世界观身份） |
| 打赏场景产品化 | 捐款按钮 | 节日/章节高潮/互动彩蛋 |
| 创作者心智 | 低价替代打赏 | 关系入场券 |

**结论**：两者都可能成立，**决定因素是 C1.5 是否守住 + 打赏场景化是否产品化**，不是分成比例。C1.5 守住时 Reviewer 成立，订阅实际是高打赏意愿漏斗。

### C3.7 C2.1 sub-commitment：订阅者专属记忆 = 平台基础设施
**Phase 2 C2.1 的运营负担被低估**：
- 若创作者手工维护订阅者专属记忆（人设卡/世界设定/订阅者身份变量/关系节点），头部 100 订阅者 × 5 记忆插槽 = 500 个定制参数，直接被拖垮
- **修正**：订阅者专属记忆必须是**平台级基础设施**（自动注入订阅标签、订阅者专属 prompt 段）
- 创作者只写"订阅者权益模板"，平台自动套用
- 否则 C2.1 身份权反变运营负担 → 创作者宁可不做

### C3.8 创作者侧漏洞清单（合并 Proposer + Reviewer 共 18 条）
**Proposer 已列**：有效粉丝定义、OC 门槛鼓励空壳、升级条件可 game、默认最低档锚定、90 天切档伤老订阅、提现腰部慢、账号异常退款、收入页缺诊断、通知不全、打赏归属链路

**Reviewer 新增 8 条**：
1. **AI 批量产图审核**：PRD 无 pHash + 内容指纹 + 风格向量三路检测；月更建议限 50 张图 / 4 章节（超出人工复审）
2. **签约独占性**：非独占但要求 Crushon 专属内容占比 ≥30%
3. **KYC 与 18+ 年龄认证**：成人收单硬要求（CCBill 2257），非可选
4. **创作者退出后订阅用户处理**：剩余月自动退款（钱包券 or 原路）+ 历史内容 30 天只读
5. **联合运营分成**：联合账号预设分成比例，后台允许 2-5 人分成配置
6. **创作者运营诊断看板**：订阅来源路径 / 档位续费率 / 内容更新后 uplift / 订阅者画像 / 流失前 3 天行为
7. **创作者→订阅者通讯**：订阅者专属推送 + 站内信 ≤4 封/月，防催更 spam
8. **休眠期保护**：主动"休眠模式" — 订阅续费暂停 + 订阅者可见休眠公告 + 内容只读访问

### C3.9 BLOCKER 级缺口标记
PRD 在补齐以下三项前**不能上线**（与 C1.6 合规并列为 pass/fail）：
- **流量扶持机制**（订阅创作者专属曝光位 / 推荐加权 / 榜单 / 更新 push）
- **关系可携带**（或替代：订阅者资产导出 + SEO + 外链位）
- **C1.5 内容形态工具**（世界观档案页 + 连载目录 + 月更节奏展示）
- **C3.7 平台级订阅者记忆基础设施**（补充 BLOCKER）
- **C3.8 第 1/3 项**：AI 批量产图审核 + KYC（成人合规硬要求）
