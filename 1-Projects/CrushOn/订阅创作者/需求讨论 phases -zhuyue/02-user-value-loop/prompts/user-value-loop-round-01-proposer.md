# 角色与任务

你是 Crushon 6.20.0 订阅创作者 PRD 评审辩论的 **提案者（Proposer）**。这是 **Phase 2 — 用户订阅价值闭环**（v2 重启后）。

## 当前阶段聚焦
**只讨论用户侧**（创作者侧 Phase 3，体系博弈 Phase 4）。
核心：用户能否形成 "首购成立 + 首月消费 + 次月续费" 的完整闭环？

## Phase 1 已确立的承诺（不得矛盾）

- **C1.1** 北极星组合：增量钱包率（排除订阅本身，及格 >-5%）+ 首购率（目标 4%）+ 新客占比（目标 >35%）
- **C1.2** 订阅与打赏偏互补但依赖分工：订阅卖"持续访问权"，打赏卖"情绪表达"；若订阅退化为美图包即转替代
- **C1.3** 毛利公式含渠道费差：订阅替代打赏可能对毛利正向（IAP 30% vs 成人收单 10-15%）
- **C1.4** 类比 = Pixiv FANBOX + DLsite：续费阈值下调至首月 40-45% / 3月 18-22% / 6月 8-12%
- **C1.5** AI 稀缺性保护需从"美图包"转向"世界观档案 + 剧情连载 + 投票"
- **C1.6** 合规可售性 = BLOCKER
- **C1.7** 打赏阶梯分成是后置方案（归属链路前置缺失）
- **C1.8** KPI 设计修正汇总

## 高风险假设（本阶段重点压测）

- **A1** 存在愿为"AI 世界观延展内容"付费的人群
- **A2** 订阅与打赏主导关系是互补（非替代）
- **A3** 可设计机制造出 AI 内容的"真稀缺感"
- **A6** 用户能理解"订阅无聊天权益"

## PRD 用户侧设计要点

**订阅入口**：仅在创作者个人页显示；开启订阅的创作者才显示

**支付**：跳转支付页，订阅走新支付主体（Payabl 2/Rapyd 2/AW 3）

**订阅管理**：个人页新增"订阅创作者"tab；列表显示作者信息、OC/相册/对话图数量、Renew/Expire 日期、Auto Renew 状态

**取消订阅**：在创作者个人页（不在订阅列表内）；文案 "Turn off auto-renewal..."

**定价**：$4.99/$14.99/$29.99/$49.99 四档，创作者四选一；90 天切档冷却，切档中断所有 Auto Renew

**订阅后可见内容**：
- OC 订阅专属相册（图/动图/视频）：入口在 OC 资料、聊天输入框、OC 列表标签；未订阅高斯模糊
- OC 对话专属图片：对话节点触发；未订阅模糊
- **⚠️ 关键事实**：订阅不含任何聊天权益；与付费内容里 OC 聊天仍需会员+credits

**通知**：订阅成功邮件；取消/到期 PRD 未明确

# 你要产出的内容（1500-2200 字，Markdown）

## 1. 用户订阅决策路径拆解（修正版）

基于"纯内容付费墙"定位重新拆解漏斗。每步含：
- 用户心里的问题
- PRD 当前答案（或缺失）
- Pixiv FANBOX / DLsite 的对应做法
- 漏斗损耗预估

步骤：看到创作者页 → 价格评估 → 专属内容预览（模糊态）→ 支付前价值确认 → 支付完成 → 首次内容消费 → 7 日回访 → 30 日复看 → 续费决策

## 2. 内容付费墙的"首月价值感知"设计评估

用户订阅 $4.99（或更高档）后进入付费墙，首月能看到什么决定了"值不值"。请评估：

- 当前 PRD 设计下，**订阅首日用户能看到的专属内容规模预估**（按创作者层级：头部/腰部/长尾）
- Pixiv FANBOX 一般档位 ¥500（~$3.5）月更  2-4 组图/1-2 篇世界观文稿。Crushon 在当前 PRD 下能给到什么量级？
- 首月内容量与 $4.99/$14.99/$29.99/$49.99 四档定价的**价值匹配度**
- 如果现在 PRD 只有"付费相册 + 对话图"两种内容形态，按 C1.5，足够支撑 40-45% 首月续费吗？

## 3. 订阅的"访问权"与"身份权"缺口分析

Pixiv FANBOX 订阅者有：世界观档案入口、创作者文字更新（blog）、订阅者徽章、订阅者评论区、投票入口。
Crushon PRD 目前只给了：相册入口、OC 对话图片入口。

**访问权差距**：
- 列出 4-6 个 PRD 缺失的"订阅者访问权"要素
- 每个要素对 C1.4 续费阈值的影响
- 实施优先级

**身份权差距**：
- 订阅者在站内是否"可见"？（是否有徽章/标记）
- 订阅时长累计、订阅者社群、订阅者评论权是否存在？
- 无身份 = 退订零成本 —— 对 C1.2 互补关系的影响

## 4. 四档定价 + 创作者四选一的心智评估

用户点到创作者页，看到只有一个价格（该创作者的选择）。请评估：
- 四档 $4.99/$14.99/$29.99/$49.99 对应的**用户心智档次**（轻度试水 / 铁粉 / 支持者 / 收藏家）
- 创作者四选一 → 用户在不同创作者看到不同价格，是否形成**价格迷惑**？
- 缺失的"阶梯权益"设计（Patreon tier 模式）：不同价格能解锁不同内容/权益？
- 90 天切档冷却 + 切档中断所有 Auto Renew：对老订阅者的伤害 + 对创作者定价试错的压制
- Round 1 用户明确"订阅和会员价无关"：这个共识下，$4.99 与会员最低档撞不撞的问题可以跳过

## 5. 续费触发机制缺口

按 C1.4，首月续费 40-45% 是生死门槛。当前 PRD 的续费触发：
- ✅ 专属相册更新
- ✅ 对话专属图片
- ❌ 到期前提醒
- ❌ 创作者新内容主动 push
- ❌ 订阅者身份感
- ❌ 月度"已解锁"消费总结
- ❌ 创作者连载承诺
- ❌ 订阅者投票参与

请评估：
- 每条缺失机制对 40-45% 续费阈值的影响幅度
- 实施优先级（P0/P1/P2）
- 哪些是 Pixiv FANBOX 类比下的最低必备要素

## 6. 用户认知风险：订阅无聊天权益是否会造成抗议

C1.2 关键：订阅 = 纯内容，聊天仍需会员+credits。这个产品定位是否会造成用户预期落差？
- 首次点击订阅的用户是否**预期订阅含聊天**？（很多竞品订阅含对话）
- 支付前是否有清晰的价值说明？
- 支付后发现"聊天还要另花钱"的挫败感风险
- 建议 PRD 补的**价值澄清机制**（入口旁边的文案、购买前的明确提示、新手引导）

## 7. iOS 端 NSFW 订阅的用户体验风险（用户要求同步上线）

因为 NSFW 必须和 SFW 同步上（C1.6 BLOCKER），iOS 端 NSFW 创作者订阅的处理方案会极大影响用户体验。请评估三种：
- **方案 A**：iOS 端完全隐藏 NSFW 创作者订阅入口（客服量爆炸风险）
- **方案 B**：展示但跳 Web 成交（审核风险）
- **方案 C**：iOS 仅展示关注/预览，订阅按钮模糊但可点击跳 Web（介于 A/B）
给出每方案的用户体验代价 + 转化损耗 + iOS 合规风险。

## 8. 用户侧漏洞清单（7-10 条）
具体可指向 PRD 原文的漏洞。每条：
- PRD 对应位置
- 漏洞描述
- 对 C1.1-C1.5 哪些承诺的影响
- 改进方向

## 9. Phase 2 评分（用 Phase 1 评估框架）
对用户侧设计打分：
- 订阅是否带来钱包外延（权重 3 生死）
- 内容付费墙续费力（权重 3 生死）
- 与打赏关系互补 vs 替代（权重 2）
- 内容形态稀缺性（权重 1）
- 价格档位一致性（权重 1）

给出 0/1/2 分 + 理由。
