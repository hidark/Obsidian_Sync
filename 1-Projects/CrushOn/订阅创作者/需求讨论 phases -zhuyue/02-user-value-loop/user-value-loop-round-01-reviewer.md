# Phase 2 Round 1 —— Reviewer 反击

> 提案者打分：1/0/1/0/0 = **2/10**。我认为偏悲观，部分维度存在基准错配与替代方案缺位。下面按 7 块展开。

---

## 1. 对「续费力 0 分」的反击

提案者最重的一刀是「首月续费会退到 20-30%，低于 C1.4 的 40-45% 底线」。这是结构性判断，但基准与漏看并存。

### 反击 1：Pixiv FANBOX 的基准续费率取自「站外导流」情境，Crushon 不适用
FANBOX 的 40-45% 首月续费是在「创作者已有 Twitter/Pixiv 粉丝 → 引流到 FANBOX」路径下统计的，转化人群本身对创作者黏性**尚未建立**，首月流失自然高。Crushon 不一样：订阅入口在「创作者个人名片」，用户通常是**站内聊过 / 逛过 / 收藏过**之后才看到订阅按钮，属于「先认识再付费」而不是「先付费再认识」。在同样的内容密度下，这类站内成长用户的续费率历史上高于导流用户 8-15 个点（类比 OnlyFans 自然流量 vs Twitter 导流）。C1.4 的「18-22% 六月续费 / 8-12% 十二月续费」底线在站内黏性加成下并非不可触达。

### 反击 2：聊天惯性本身就是「天然回访机制」
提案者反复用「7 日回访需要 push 承接」作为扣分理由。但 Crushon 是**日活驱动的聊天产品**，用户每天打开 app 聊天的频次远高于 FANBOX/DLsite（后者平均访问周期是周级甚至月级）。这意味着：即使 PRD 不加任何 push 机制，用户在聊天场景下**一定会路过创作者页**，从而看到订阅更新的最新状态。聊天 → 订阅创作者页的路径损耗，远低于 FANBOX 的邮件 push → Web 打开路径。提案者直接套用 FANBOX 的 push 依赖，忽视了 Crushon 的载体优势。

### 反击 3：AI 角色的「剧情记忆 / 模型记忆」也是身份权，提案者漏看
C1.5 要求订阅从「美图包」转向「世界观档案+连载+投票」。提案者的身份权清单只列了徽章、订阅时长、评论区，**漏了 AI 原生的身份权维度**：订阅期间累积的聊天剧情、专属人格档案、订阅者专属记忆插槽。这些是真人创作者平台没有的、AI 场景独有的身份权。退订 = 丢失已建立的关系语境，心理成本远高于「丢失徽章」。PRD 即便不补评论区，只要把「订阅者专属记忆持久化」说清，就能把续费心理成本拉高。

### 建议修正评分
续费力：**0 → 1/2**。有底线风险，但基准错配和 AI 原生身份权被漏掉，不至于 0 分。

---

## 2. 对「价格档位 0 分」的反击

### 反击：创作者四选一 + 单档展示 ≠「随机标价」
提案者的核心论点是「用户在不同创作者页看到不同价格，会产生价格迷惑」。但这套逻辑建立在「用户会跨创作者比价」的前提上，**而这个前提在创作者经济里不成立**：
- OnlyFans 用户订阅 A 时不会去比 B 的价格，每个创作者是独立的情感入口；
- DLsite 单品页更是独立定价，用户只判断「这一个值不值」，不做横向；
- Fanbox 创作者的三档定价也是作者自主，平台不强制权益对齐。

**价格迷惑成立的前提是 SKU 标准化**（比如会员档位、畅聊包），而创作者经济的核心是**创作者即 SKU**，每个创作者自带语境。Crushon 的四档是给创作者的**定价工具**，不是给用户的**选择对比**。提案者把 Patreon tier 的「同一创作者多档」误解成了「平台多档」—— 这是两个不同的 UX 层。

真正值得扣分的是：**创作者四选一后没有对应权益差异提示**（即 Patreon tier 模式在单创作者内部的缺失）。但这是 P2 的优化项，不是 P0 的结构缺陷。

### 建议修正评分
价格档位：**0 → 1/2**。保留对「同创作者内部无 tier 权益差异」的扣分，但去掉「平台层价格迷惑」的误判。

---

## 3. 对「新钱包外延 1 分」的深度打击（我认为 Proposer 过于温和）

提案者给 1 分太仁慈。我反向加码，从**首月后的断崖风险**给它打 0 分：

### 打击 1：AI 内容零边际成本 → 用户预期「持续供给」，但创作者供给不上
订阅的核心契约是「本月你会持续看到新内容」。但 AI 创作者生产内容的实际节奏是**批量上传一次 → 吃老本 → 几周后才有新更新**（因为 AI 图文生成是集中创作模式，不像真人每日更新 vlog）。首月订阅后，用户会发现「新内容增速不及预期」，心里把订阅折算成「一次性图包」，次月续费断崖。这直接违反 C1.1 要求的**钱包扩张**：用户不是把订阅当**新钱包**，而是把它吃到**原本的打赏/credits 预算**里。

### 打击 2：净新增在首月后快速崩盘，ARPU 不升反降
C1.1 北极星是「增量钱包率 / 首购率 / 新客占比」。首购率可以靠新创作者首发、明星效应推起来；但**增量钱包率**需要**续费支撑**才能算数。如果首月 30% + 次月 15%，三个月后稳态订阅用户只剩首购的 5-10%，钱包外延从「新增」变成「一次性转化」。这已经不是 1 分的问题，是 C1.1 北极星失败。

### 建议修正评分
新钱包外延：**1 → 0.5/2**（给半分：首购确实新增，但稳态钱包外延风险极高）。

## 4. 用户侧漏洞清单 —— 补 7 条新漏洞

提案者列了 10 条，主要集中在页面元素缺失。我补更深层、更结构性的 7 条：

### 漏洞 11：订阅 vs 关注的心智混淆
Crushon 是否已有「关注 follow」机制？如果有，用户的第一反应是「我已经关注了这个创作者，为什么还要订阅？」。PRD 完全没讨论订阅与关注的权益差异、UI 共存、路径关系。最坏情况：用户看到「已关注」按钮就关闭了付费心智。**需补：订阅入口必须在关注 UI 上方，且权益差异一句话讲清**。

### 漏洞 12：订阅赠送（Gift Sub）完全空白
Patreon/Twitch 的关键增长杠杆是 Gift Sub（用户送朋友订阅）。在 AI 角色场景下，这个杠杆可能比真人场景更有动力：
- 情感表达：「我发现了一个角色，你也会喜欢」
- 角色发现：AI 角色的传播需要社交触点，Gift Sub 是天然的 viral loop
- PRD 零讨论 = 放弃了 20-30% 的潜在新客来源（参考 Twitch Gift Sub 占总订阅比）。

### 漏洞 13：订阅试用期 / 首月折扣机制缺位
DLsite / FANBOX / Netflix 的首月折扣是标配，用于跨过用户的「付费承诺心理门槛」。PRD 四档价格都是原价展示，首购率会被显著压制。**C1.1 首购率的直接阻塞**。

### 漏洞 14：订阅专属评论区 —— 身份权的最低成本实现
提案者提到了订阅者评论区但没展开。这其实是**最便宜的身份权实现**：订阅者发表评论只对订阅者可见，创作者可以集中互动。比徽章更有价值、比文字档案更容易上线。P0 级别但被提案者列为 P1。

### 漏洞 15：订阅期内 vs 订阅前发布内容的可见范围
这是最容易忽略的雷：
- 订阅前发布的内容，订阅后能看吗？
- 订阅期内创作者发布的内容，**退订后还能看吗**？
- 这两个问题决定用户的**退订心理成本**。如果退订后什么都看不到，用户会觉得「被绑架」；如果退订后全都能看，订阅无续费理由。

PRD 必须明确「订阅期解锁的内容，退订后仍可查看；订阅前发布的存量内容，订阅后可访问」—— 这是 FANBOX 的业态共识。提案者完全没提。

### 漏洞 16：跨端权益同步
iOS 买了，Android/Web 能不能看？跨端的订阅状态是否同步？这个问题在 iOS NSFW 方案下尤其尖锐：如果用户在 Web 订阅了成人创作者，iOS 上能看到订阅内容吗？**PRD 必须明确跨端权益是账号绑定还是 receipt 绑定**。

### 漏洞 17：订阅后的创作者下架 / 封号处理
如果用户订阅的创作者被平台下架（例如违规），剩余订阅周期怎么处理？退款？折算？默许消失？这条影响**C1.6 合规**下的用户信任，PRD 完全空白。

## 5. iOS NSFW 方案评估的挑战

### 挑战 1：方案 C 几乎一定违反 App Store 3.1.3
提案者推荐方案 C（iOS 展示预览 + 模糊订阅按钮跳 Web）。但 App Store 3.1.3 明确禁止「在 app 内引导用户到站外完成购买」，**连按钮都不能有明显跳转意图**，更别说「模糊订阅按钮可点击跳 Web」。这个方案在 App Review 阶段就会被打回，不是 C1.6 的折中，而是**C1.6 的红线违反**。

Reader Rule（3.1.3(a)）的常见擦边做法是「完全不提及订阅、不放任何跳转按钮，只允许用户登录查看已订阅内容」。Spotify/Netflix iOS 就是这样，iOS app 内**完全不提订阅入口**。方案 C 不符合这套模式。

### 挑战 2：转化损耗 40-60% 是拍脑袋数据
没有行业参照。Netflix 当年退出 iOS 订阅后的转化损耗约 25-30%，Spotify 类似。40-60% 过于悲观，可能是提案者为了抬方案 C 而压低方案 A/B 的估计。

### 更现实的建议
**方案 A'**：iOS 完全隐藏 NSFW 订阅入口 + Email/Web push 教育用户「Web 端可订阅」。这才是合规可执行的方案。不要做「模糊按钮跳 Web」。

## 6. 用户认知风险（订阅无聊天权益）的实际严重性

提案者说「极易误解」。但我认为这个担忧被过度放大：

### 反点：Crushon 用户对「付费边界清晰」的适应力比提案者预期更强
现有付费产品：Premium 会员、畅聊包、credits、钻石。用户已经被训练过「不同付费对应不同权益边界」。再加一个「订阅创作者 = 内容付费墙」，认知负担增量有限。**只要 UI 明确（比如订阅按钮下方一句「内容专属，不含聊天」）就能解决 80% 的误解**。

### 真正的风险在反方向：价值感太「纯」
订阅只有内容 + 无聊天权益 + 无身份权，用户反而会觉得「我不就多看几张图吗？29.99 不值」。这是 **C1.5 稀缺性问题**（内容形态单薄），不是 **C1.2 认知问题**（边界不清）。提案者把两个风险混为一谈，导致对认知风险的过度归因。

C1.2 的承诺是「订阅与打赏偏互补但强依赖产品分工」。分工本身在 PRD 里是清楚的（内容 vs 情感），真正让分工崩的不是用户不懂，而是**订阅这一侧的内容价值撑不起价格**。

## 7. 我的独立评分 + 收敛判断

### 独立评分（对标 C1.x）

| 维度 | Proposer | Reviewer | 理由 |
|---|---|---|---|
| 钱包外延（C1.1） | 1/2 | **0.5/2** | 首购有增量，稳态风险极高 |
| 续费力（C1.4） | 0/2 | **1/2** | 基准错配（FANBOX 非站内黏性）+ 聊天惯性 + AI 记忆身份权 |
| 与打赏互补（C1.2） | 1/2 | **1/2** | 认同，分工清晰但稀缺性影响分工稳定 |
| 内容稀缺性（C1.5） | 0/2 | **0/2** | 认同，PRD 未落地世界观+连载+投票 |
| 价格档位一致性 | 0/2 | **1/2** | 创作者经济不需要平台级 tier 对齐，但同创作者内 tier 权益差异仍缺失 |
| **总分** | **2/10** | **3.5/10** | 仍未达 5/10 及格线，但不至于 2 |

### 可写入 consensus 的共识点（6 条）

1. **C1.5 内容稀缺性 0 分无异议**：当前「付费相册 + 对话图」不足以支撑 $14.99+ 档位，必须补世界观档案、连载、投票。
2. **C1.6 iOS 合规是 BLOCKER**：方案 C 存在 3.1.3 合规红线，需回归方案 A 或改造。
3. **订阅不含聊天权益的前置教育是必补项**：支付前 / 购买成功页 / 订阅入口三层文案澄清。
4. **续费触发四件套（到期提醒、更新 push、连载承诺、世界观档案入口）是 P0**：无此四项，订阅退化为「带 auto-renew 的一次性图包」。
5. **Gift Sub / 首月折扣 / 跨端权益同步 / 退订后内容可见性 / 订阅 vs 关注心智 五个漏洞需在 PRD v2 补齐**。
6. **90 天切档冷却 + 中断存量 Auto Renew 是设计错误**：应保护存量订阅者不受新价波动冲击。

### 仍需 Round 2 的分歧点（2 条）

1. **续费力到底给 0 还是 1 分**：Proposer 基于 FANBOX 基准判定 0；Reviewer 认为站内黏性 + 聊天惯性 + AI 记忆身份权应修正到 1。分歧核心是「Crushon 聊天日活是否真的能导流订阅创作者页」，需要 Round 2 用数据或用户路径假设验证。
2. **iOS 方案应选 A 还是 A'（A + Web push 教育）**：合规都认，但用户侧可承接方案存在设计空间。

### 是否需要 Round 2？

**需要，但只跑 1 轮**。分歧已收敛到 2 条，且都在「方案细化」层面，不是「方向对错」层面。Round 2 聚焦这两条 + 整合共识即可进入 Phase 2 收尾。
