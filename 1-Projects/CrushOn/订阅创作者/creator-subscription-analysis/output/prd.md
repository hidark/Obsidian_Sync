# CrushOn 创作者订阅功能 PRD

**版本**：1.0  
**状态**：待评审  
**完成日期**：2026-04-20  
**分析范围**：基于"订阅创作者.pdf"的产品设计评估，覆盖订阅机制、创作者激励、订阅者体验、平台商业模式、三方利益平衡、优化建议六个维度。

---

## 一、产品概述

### 1.1 功能定位

CrushOn创作者订阅功能允许平台上的AI角色创作者向订阅者提供付费专属内容，构建创作者→订阅者→平台的三方价值循环。

### 1.2 核心设计原则

- **单层级订阅**：创作者自主定价（$4.99-$49.99/月），不预设多层级套餐
- **全量历史访问**：订阅者在订阅期间可访问创作者所有历史内容，订阅结束后立即失去访问权限
- **透明续费**：自动续费三要素（明确告知、清晰取消入口、提前24小时通知）
- **25%平台抽成**：创作者实得75%，处于行业可接受范围

### 1.3 整体评估结论

**有条件可行**。核心机制设计方向正确，不存在需要重新设计的根本性问题。主要问题集中在执行层面：工具缺失、流程矛盾、信息不透明、支付成功率低。完成6项MVP P0条件后可上线，Year 1内补足7项P1功能。

## 三、核心功能规格

### 3.1 订阅机制

**订阅创建**：
- 创作者自主设定订阅价格（$4.99-$49.99/月）
- 订阅者在创作者主页完成订阅，支付后立即获得全量历史内容访问权限
- 订阅确认页和邮件确认均需明确告知自动续费规则

**价格变更处理（路径C，MVP）**：
- 创作者变更订阅价格时，立即通知所有现有订阅者（含新价格、生效时间、重新订阅入口）
- 通知后7天内，订阅者可申请按比例退款
- 提供简化重新订阅流程（预填支付信息，2步以内完成）
- 90天内不得重复变更价格

**价格变更处理（路径A，Year 1 Q2-Q3）**：
- 创作者涨价时，提前14天通知现有订阅者
- 订阅者可选择"锁定当前价格续费一个周期"（仅一次机会）
- 一个周期后统一执行新价格

**支付失败处理**：
- 支付失败后提供7天宽限期
- 首次失败：保留全量访问；连续2次失败：限制为50%访问量；连续3次失败：取消宽限期资格
- 宽限期结束后重新订阅需等待7天冷却期

**内容访问控制**：
- 订阅期间：全量历史内容访问（包括订阅前发布的内容）
- 订阅结束后：立即失去访问权限
- Premium Archive例外：创作者可将部分内容设为PPV单独销售，PPV内容占比不超过总内容的30%

### 3.2 创作者工具（MVP必须实现）

| 工具 | 功能描述 | 优先级 |
|------|---------|-------|
| 收入仪表盘 | 总收入、订阅者数、增长趋势、收入明细 | P0 |
| 订阅者列表 | 查看、筛选、导出订阅者信息 | P0 |
| 订阅内容标记 | 标记哪些内容为订阅专属，设置内容可见性 | P0 |
| 提现入口 | 绑定支付账户、发起提现、查看提现记录 | P0 |
| 收入趋势分析 | 历史收入趋势、订阅者增长分析 | P1（Q2） |
| 群发消息 | 向所有订阅者发送消息 | P1（Q3） |
| 促销活动设置 | 设置限时折扣、新订阅者优惠 | P1（Q3） |

### 3.3 订阅者体验

**订阅前**（MVP必须实现）：
- 创作者主页展示：专属内容数量、最近更新时间、当前订阅者数量
- 订阅价格和权益说明清晰可见
- 提供部分免费内容预览（降低订阅决策门槛）

**订阅后**：
- 专属相册、专属对话图片、全量历史内容访问
- 内容更新通知（P1，Q3）
- 创作者私信（P1，Q3）
- 订阅者专属群组（P1，Q3）

**续费管理**：
- 账户设置中提供一键取消入口
- 续费前至少提前24小时发送提醒通知
- 支持订阅暂停（P1，Q3，每年最多2次，每次最长30天）

## 四、平台商业模式

### 4.1 收入模型

- **平台抽成**：25%（创作者实得75%）
- **ARPU基准**：$9（保守基准），区间$8-$12
- **Year 1目标**：5,000+活跃订阅者，月度GMV≥$50K
- **盈亏平衡**：20,000活跃订阅者（Year 2目标）

### 4.2 Year 1可行性三个必要条件

1. **ARPU验证**：实际ARPU≥$9（MVP上线后首月监控，若<$9立即启动成本优化）
2. **C3.2矛盾解决**：MVP上线前完成路径C，将额外流失率从10-15%降至5-8%
3. **支付成功率提升**：从当前35-45%提升至50%+（Year 1目标60%+）

### 4.3 抽成稳定承诺

- 上线后12个月内抽成率锁定25%，不单方面上调
- 调整抽成率需提前90天通知创作者
- 承诺形式：在创作者协议中明确写明，并在创作者后台公开展示

## 五、Year 1优化路线图（P1任务）

| 时间 | 任务 | 成本 | 预期收益 |
|------|------|------|---------|
| Q1（上线即承诺） | 抽成稳定承诺（12个月锁定25%） | $1K | 创作者长期信任 |
| Q2 | 路径A（14天通知+一周期锁价） | $25K-$35K | 额外流失率降至2-3% |
| Q2 | 冷启动支持（流量扶持+入驻引导） | $8K-$13K | 新创作者留存率提升20-30% |
| Q2 | 创作者停更补偿（30天无更新→暂停新订阅+通知） | $10K-$15K | 订阅者满意度提升 |
| Q3 | 互动通道（创作者私信+订阅者群组） | $20K-$25K | 订阅者LTV提升15-20% |
| Q3 | 订阅纠纷仲裁机制 | $8K-$12K | 平台信任度提升 |
| Q3 | 订阅暂停功能（2次/年，30天/次） | $5K-$8K | 12个月留存率提升10-15% |

**Year 1总成本**：$77K-$109K

## 七、风险与缓解措施

| 风险 | 严重程度 | 缓解措施 |
|------|---------|---------|
| Year 1目标未达成（余量仅8.4%） | 高 | 三个必要条件监控+应急预案 |
| 竞品在6-12个月内推出创作者变现 | 高 | 加速P0工具上线，建立工具完善度壁垒 |
| 创作者流失率超过60%（悲观情景） | 高 | 冷启动支持+抽成稳定承诺+停更补偿机制 |
| ARPU偏向$8（下行风险） | 中高 | 首月监控+成本优化预案 |
| Year 3平台强势化（抽成上调风险） | 中 | 12个月抽成稳定承诺+创作者议价保护条款 |
| Premium Archive过度PPV化 | 中 | PPV内容占比上限30%+违规警告机制 |

## 八、待明确事项（运营手册）

1. **纠纷仲裁边界**："主观不满"与"客观违约"的边界需在运营手册中进一步细化（如创作者承诺更新频率但未达标的处理标准）
2. **冷启动流量扶持细则**：新创作者推荐位的算法权重、扶持期限、退出条件
3. **停更补偿的活跃度阈值**：30天无更新的定义（是否包含回复订阅者消息？）
4. **支付渠道扩展时机**：Year 1 Q2评估是否需要增加加密货币等渠道
